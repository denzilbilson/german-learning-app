## Phase 3: Text Analysis — Done

### New files

**`server/services/claude-service.js`**
- Wraps `@anthropic-ai/sdk`, reads `ANTHROPIC_API_KEY` from env
- `callClaude(systemPrompt, userMessage, maxRetries=2)` → parsed JSON
- Strips markdown code fences from responses before parsing
- Retries up to 2× on `SyntaxError` (malformed JSON), throws immediately on API/network errors

**`server/prompts/analyze-text.js`**
- Detailed system prompt calibrated for B1→B2 learner
- Requests structured JSON: `translations` (line-by-line), `vocabulary` (word + literal/intended meaning + POS + case examples + CEFR level), `phrases` (idioms/collocations), `grammar` (2–5 rules with plain-language explanations)
- Enforces JSON-only output

**`server/routes/analyze.js`**
- `POST /api/analyze` — calls Claude, saves result to `data/analysis/analysis-<timestamp>.json`, returns full result
- `POST /api/analyze/add` — duplicate-checks against existing vocab/phrases (case-insensitive), adds new entries to markdown files, returns `{ added, skipped }` counts
- `GET /api/analysis` — lists saved analyses with metadata (vocab count, phrase count, preview)
- `GET /api/analysis/:filename` — returns a specific saved analysis; filename sanitized with regex to prevent path traversal

**`client/src/components/AnalysisReader.jsx`** (360 lines)
- `TranslationSection` — two-column German/English table, alternating row backgrounds
- `VocabSection` — sortable table with checkboxes, expandable rows for case examples, POS badges (colour-matched to Vocabulary page), CEFR level badges
- `PhrasesSection` — card-per-phrase with checkboxes, level badge
- `GrammarSection` — accordion with numbered topics, first one open by default

**`client/src/pages/TextAnalysis.jsx`** (280 lines)
