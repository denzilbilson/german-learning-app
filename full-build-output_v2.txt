## Review Pass 1 — Correctness: Findings

### Bugs Fixed

**1. `client/src/services/api.js:84` — Double `res.json()` stream exhaustion**
```js
// BEFORE (broken):
try { msg = (await res.json()).message || (await res.json()).error } catch { ... }

// AFTER (fixed):
try { const d = await res.json(); msg = d.message || d.error } catch { ... }
```
`Response.body` is a one-shot readable stream. The second `await res.json()` would throw "body already used" whenever `.message` was falsy (i.e., the server returned `{ error: '...' }` without a `message` key — the standard shape used by every route). Anki export failures would silently show a blank error message.

**2. `client/src/pages/Import.jsx:142` — `useState` used as an effect trigger**
```js
// BEFORE (broken):
useState(() => { onMappingChange(initialMapping) })

// AFTER (fixed):
useEffect(() => { onMappingChange(initialMapping) }, [])
```
`useState` initializers run during render. Calling `onMappingChange` (which is the parent's `setMapping`) during `MappingStep`'s render caused React's *"Cannot update a component while rendering a different component"* warning, which indicates a possible tear in React's render/commit cycle. Changed to `useEffect` (runs post-render) and replaced the unused `useCallback` import with `useEffect`.
